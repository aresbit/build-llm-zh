# 第2章: 处理文本数据 (Working with Text Data)

深度神经网络，包括大语言模型（LLM），无法直接处理原始文本。文本是离散的类别数据，与神经网络的数学运算不兼容。因此，我们必须将词表示为连续值向量。将数据转换为向量格式的概念称为**嵌入（embedding）**。数据嵌入不仅适用于文本：音频、图像、视频都有各自的嵌入模型，不同模态的数据需要不同的嵌入模型，一种模型不能跨模态通用。

本章我们将逐步走完为 LLM 准备输入嵌入的全流程：分词（tokenization）、构建词表、将 Token 转换为 Token ID、引入特殊上下文 Token、使用 BPE 算法处理未登录词、通过滑动窗口生成训练样本对、创建 Token 嵌入以及位置编码。

---

## 2.1 词嵌入 (Word Embedding)

嵌入的本质，是从离散对象（词、图像、文档等）到连续向量空间的一个映射。我们将其形式化：

### 嵌入的形式化定义

设 $$V$$ 为有限词表，$$\vert V\vert  = v$$。一个词嵌入是一个映射：

$$W: V \to \mathbb{R}^d$$

其中 $$d$$ 是嵌入维度。$$W$$ 可以表示为一个形状为 $$v \times d$$ 的矩阵（查找表）。对于任意词 $$w \in V$$，其嵌入向量为 $$W$$ 中该词对应的行向量：

$$\mathbf{e}_w = W[w] \in \mathbb{R}^d$$

在 PyTorch 中，`torch.nn.Embedding(v, d)` 本质上就是维护了这样一个 $$v \times d$$ 的可训练权重矩阵。给定一个 Token ID $$i$$，嵌入操作等价于从权重矩阵中取出第 $$i$$ 行。

### 嵌入维度的权衡

嵌入维度 $$d$$ 的选择是一个性能与效率之间的核心权衡：

$$
\begin{aligned}
\text{参数量} &= v \cdot d \\
\text{表达能力} &\propto d \quad (\text{更高维度能捕捉更细腻的语义关系}) \\
\text{计算开销} &\propto d \quad (\text{矩阵乘法的 FLOPs 随 } d \text{ 线性增长}) \\
\text{内存占用} &\propto d \quad (\text{每层激活值存储})
\end{aligned}
$$

以 GPT-2 和 GPT-3 为例：
- GPT-2 Small（117M 参数）：$$d = 768$$，$$v = 50257$$，嵌入层参数约 $$50257 \times 768 \approx 38.6\text{M}$$
- GPT-3 175B：$$d = 12288$$，嵌入层参数约 $$50257 \times 12288 \approx 617.6\text{M}$$

嵌入层参数量占模型总参数量的比例大约是：

$$\frac{v \cdot d}{P_{\text{total}}} \approx \frac{50257 \cdot d}{12 \cdot n_{\text{layers}} \cdot d^2} = \frac{50257}{12 \cdot n_{\text{layers}} \cdot d}$$

其中 $$n_{\text{layers}}$$ 是 Transformer 层数。由此可见，随模型规模增大（$$n_{\text{layers}}$$ 和 $$d$$ 均增大），嵌入层参数占比逐渐降低，但在小模型中嵌入层占比不容忽视。

### Word2Vec 与可训练嵌入

Word2Vec 是经典的词嵌入方法，其核心思想是"分布假说"：出现在相似上下文中的词具有相似的含义。在 LLM 中，我们通常不依赖预训练的 Word2Vec 嵌入，而是在训练过程中从零开始学习嵌入。这样做的好处是嵌入能够针对特定任务和数据分布进行联合优化，获得"上下文化"的表示（参见第 3 章注意力机制）。

## 2.2 分词 (Tokenization)

分词是将原始文本拆分为 Token 序列的预处理步骤。Token 可以是词、子词或字符级别。常见的分词粒度包括：

- **词级分词 (Word-level)**：按空格和标点分割。实现简单，但词表会随语料规模增大而膨胀，且无法处理新词（未登录词，OOV）。
- **子词分词 (Sub-word)**：通过统计方法将常见字符组合合并为子词单元。BPE 是代表方法。
- **字符分词 (Character-level)**：以单个字符为 Token。词表极小（通常 < 256），但序列长度急剧增加，使长距离依赖更难捕捉。
- **字节级分词 (Byte-level)**：以 UTF-8 字节为基本单元。词表仅 256，可以编码任意文本，但对英语以外的语言，序列长度膨胀更为严重。

本节我们先用正则表达式实现一个简单的词级分词器，以建立直观理解。对输入文本：

```
"Hello, world. Is this-- a test?"
```

使用正则表达式 `([,.:;?_!"()']|--|\s)` 进行切分并去除空白符后得到：

```
['Hello', ',', 'world', '.', 'Is', 'this', '--', 'a', 'test', '?']
```

将此分词器应用于 Edith Wharton 短篇小说 *The Verdict*（共计 20479 个字符），得到 4690 个 Token（去除空格）。

## 2.3 从 Token 到 Token ID

有了 Token 列表后，我们需要构建一个**词表**，将每个唯一的 Token 映射为一个唯一的整数 ID。构建流程为：

1. 收集训练数据中所有唯一 Token
2. 按字母序排序
3. 为每个 Token 分配一个从 0 开始的连续整数 ID

设词表为 $$\mathcal{V}$$，对于 Token $$t$$，其 Token ID 为 $$\text{id}(t)$$，我们同时维护一个反向映射 $$\text{token}(\text{id})$$ 将整数还原为 Token：

$$\mathcal{V}: \text{Token} \mapsto \{0, 1, \dots, \vert \mathcal{V}\vert -1\}$$

词表大小 $$\vert \mathcal{V}\vert $$ 直接影响：
- 每个 Token ID 的表示宽度（$$\lceil \log_2 \vert \mathcal{V}\vert  \rceil$$ 位）
- 嵌入层的参数量（$$\vert \mathcal{V}\vert  \times d$$ 个参数）
- 对未登录词的处理策略

对 *The Verdict* 文本，唯一 Token 数为 1130。实现一个完整的 `SimpleTokenizerV1` 类，包含 `encode(text)` 和 `decode(ids)` 方法。`encode` 方法先将文本切分为 Token 列表，再通过词表查表转为整数；`decode` 方法做反向映射。

然而，当遇到训练集中不存在的词时（如 "Hello"），分词器会抛出 `KeyError`。这凸显了词表覆盖度的重要性——真实 LLM 训练需要使用海量多样化语料，并采用更先进的分词策略。

## 2.4 特殊上下文 Token

为解决上述问题，我们引入特殊 Token：

- **`<|unk|>`**：代表未知词（Out-of-Vocabulary）。当分词器遇到词表中不存在的词时，将其替换为此 Token。
- **`<|endoftext|>`**：文档分隔符。当训练语料由多个独立文档拼接而成时，插入此 Token 使模型理解文档边界，避免跨文档的虚假关联。

常见特殊 Token 还包括：

- **[BOS]**（序列开始）：标记一段文本的起始，提示模型内容开始。
- **[EOS]**（序列结束）：标记文本结束，与 `<|endoftext|>` 功能类似。
- **[PAD]**（填充）：批处理时用于将不等长序列填充至统一长度。

GPT 系列模型的 Tokenizer 仅使用 `<|endoftext|>`，它兼具分隔符和填充的功能（配合注意力掩码，填充 Token 不会影响计算）。更重要的是，GPT 不使用 `<|unk|>` Token——这得益于 BPE 算法的独特设计。

## 2.5 字节对编码 (Byte Pair Encoding, BPE)

BPE 是一种数据驱动的子词分词算法，被 GPT-2、GPT-3 和 ChatGPT 原始模型广泛采用。其核心优势在于**能够处理任意未知词**而无需引入 `<|unk|>` Token。

### BPE 算法详解

BPE 从字符级别出发，通过迭代合并最高频的 Token 对来构建词表。具体步骤如下：

**初始状态**：将训练语料中的每个词表示为字符序列，末尾添加一个特殊的词结束符（如 `</w>`，GPT 的 BPE 中不使用显式词结束符，而是通过空格隐式区分词边界）。

**第 1 步（初始化）**：将所有字符（包括空格、标点等）加入基础词表。

**第 2 步（迭代合并）**：重复以下过程直到达到目标词表大小 $$V_{\text{target}}$$：

1. 统计当前语料中所有相邻 Token 对 $$(a, b)$$ 的频数。
2. 选择频数最高的 Token 对 $$(a^*, b^*)$$。
3. 将合并后的新 Token $$a^*b^*$$ 加入词表。
4. 在语料中将所有出现的 $$a^* b^*$$ 替换为 $$a^*b^*$$。

**第 3 步（终止）**：词表达到目标大小，停止合并。最终词表包含基础字符和所有合并产生的子词。

### 一个工作实例

假设语料仅由两个词组成：`"low"`（出现5次）和 `"lower"`（出现2次），目标词表大小 $$V_{\text{target}} = 10$$。

**初始词表**（包含基本字符）：`{l, o, w, e, r}`，共 5 个 Token。

语料表示（字符级）：
- `l o w` (×5)
- `l o w e r` (×2)

**第 1 轮合并**：统计相邻对频数：
- `(l, o)`: 7
- `(o, w)`: 7
- `(w, e)`: 2
- `(e, r)`: 2

最高频对为 `(l, o)`（或 `(o, w)`，同为 7。按先到先得选择 `(l, o)`）。

合并为 `lo`，词表更新为 `{l, o, w, e, r, lo}` (6)。

**第 2 轮**：语料变为：
- `lo w` (×5)
- `lo w e r` (×2)

统计：`(lo, w)`: 7, `(w, e)`: 2, `(e, r)`: 2。合并 `(lo, w)` → `low`，词表 (7)。

**第 3 轮**：语料：
- `low` (×5)
- `low e r` (×2)

统计：`(low, e)`: 2, `(e, r)`: 2。合并 `(low, e)` → `lowe`，词表 (8)。

**第 4 轮**：语料：
- `low` (×5)
- `lowe r` (×2)

统计：`(lowe, r)`: 2。合并 `(lowe, r)` → `lower`，词表 (9)。

**第 5 轮**：语料：
- `low` (×5)
- `lower` (×2)

此时只剩两个独立 Token，没有相邻对可合并。如果未达到目标词表大小，则训练结束。最终词表：`{l, o, w, e, r, lo, low, lowe, lower}` (9)。

### 词表大小与压缩率的权衡

压缩率定义为原始文本的字符数除以 Token 数：

$$\text{压缩率} = \frac{\text{字符总数}}{\text{Token 总数}}$$

设词表大小为 $$V$$，压缩率为 $$C(V)$$，则：

- 当 $$V$$ 较小时，Token 倾向于为单字符，压缩率接近 1（无压缩）。
- 当 $$V$$ 增大时，更多常见的子词被合并为单个 Token，压缩率随之增大。
- 当 $$V$$ 趋近于语料中所有可能子词的数量时，压缩率达上界。

GPT-2 的 BPE 词表大小为 $$V = 50257$$，对于英文文本，典型压缩率约为 3-4（每 3-4 个字符构成一个平均 Token）。$$V$$ 的选取需权衡：过小则序列变长、效率降低；过大则嵌入层参数过多、显存压力增大。

BPE 相比词级分词的关键优势：**任何未知词都可以被分解为已知子词或字符的组合**。例如，即便是训练集中从未出现过的随机字符串 `"Akwirwier"`，BPE 也能将其拆分为已知子词，无需 `<|unk|>` Token。

OpenAI 的开源库 `tiktoken` 提供了高效的 BPE 实现（基于 Rust 编写），通过 `tiktoken.get_encoding("gpt2")` 即可获取与 GPT-2 一致的 Tokenizer。

## 2.6 滑动窗口数据采样 (Sliding Window Data Sampling)

LLM 的训练目标是**预测下一个 Token**。给定一段文本，我们需要从 Token 序列中构造输入-目标对：输入 $$x$$ 是前文上下文，目标 $$y$$ 是紧接其后的 Token。

### 样本数量的推导

设 Token 化后的文本长度为 $$N$$，上下文窗口大小为 $$W$$（即每个输入样本包含 $$W$$ 个连续 Token），步长为 $$S$$（滑动窗口每次移动的 Token 数）。则从该文本中可生成的样本数 $$M$$ 为：

$$M = \left\lfloor \frac{N - W}{S} \right\rfloor + 1$$

推导：第一个样本的起始位置为 0，最后一个样本的起始位置 $$k$$ 满足 $$k + W \leq N$$，即 $$k \leq N - W$$。步长为 $$S$$ 意味着起始位置构成等差数列 $$0, S, 2S, \dots, \lfloor (N - W)/S \rfloor \cdot S$$，项数即为 $$M$$。

特别地，当 $$S = 1$$ 时（最大重叠采样）：

$$M = N - W + 1$$

当 $$S = W$$ 时（无重叠采样）：

$$M = \left\lfloor \frac{N}{W} \right\rfloor$$

步长选择的影响：
- $$S = 1$$（高重叠）：生成最多样本，但相邻样本高度相似，可能增加过拟合风险。
- $$S = W$$（无重叠）：每个 Token 仅被一个样本覆盖，数据利用率较低。
- 实践中通常取 $$1 < S < W$$ 作为折中（如 $$S = W/2$$）。

以 *The Verdict* 的 BPE Token 化结果为例，$$N = 5145$$，$$W = 4$$，$$S = 1$$，则 $$M = 5145 - 4 + 1 = 5142$$。

### 批量化数据加载

通过 PyTorch 的 `Dataset` 和 `DataLoader` 实现高效的数据加载器。关键参数：
- `max_length`：输入上下文窗口大小 $$W$$
- `stride`：滑动步长 $$S$$
- `batch_size`：每个批次的样本数
- `shuffle`：是否打乱样本顺序

`GPTDatasetV1` 将整个文本 Token 化后，通过循环生成输入块 `input_chunk` 和目标块 `target_chunk`（后者为前者右移一个位置），分别存入 `input_ids` 和 `target_ids`。

当 `batch_size = 8`、`max_length = 4`、`stride = 4` 时，输出为 $$8 \times 4$$ 的输入张量和 $$8 \times 4$$ 的目标张量。

## 2.7 Token 嵌入 (Token Embeddings)

Token 嵌入层将离散的 Token ID 映射为连续的向量表示。在 PyTorch 中：

```python
embedding_layer = torch.nn.Embedding(vocab_size, output_dim)
```

本质上，这是一个形状为 $$\text{vocab\_size} \times \text{output\_dim}$$ 的可学习权重矩阵 $$W_e$$。给定输入 Token ID 序列 $$\mathbf{t} = [t_1, t_2, \dots, t_L]$$，嵌入操作等价于查表：

$$\mathbf{x}_i^{\text{token}} = W_e[t_i], \quad \mathbf{x}_i^{\text{token}} \in \mathbb{R}^d$$

对于一批 8 个样本、每个 4 个 Token 的输入（$$8 \times 4$$ 张量），经过 $$d = 256$$ 嵌入层后，输出为 $$8 \times 4 \times 256$$ 张量。

一种等价看待嵌入层的方式是从独热编码的角度理解：将 Token ID 先做 One-Hot 编码得到一个 $$v$$ 维稀疏向量 $$\mathbf{1}_{t_i} \in \{0,1\}^v$$，然后乘以嵌入矩阵 $$W_e$$：

$$\mathbf{x}_i^{\text{token}} = \mathbf{1}_{t_i}^T W_e$$

这正是矩阵乘法在查找表中的高效等价形式。嵌入层可以视为一个神经网络层，其参数通过反向传播进行优化。

## 2.8 位置编码 (Positional Encoding)

### 为什么需要位置编码

LLM 的自注意力机制（详见第 3 章）具有置换等变性——对输入序列任意重排，注意力计算的输出也会相应重排，但其值不变。换言之，自注意力机制本身不感知 Token 的顺序。然而，语言中的词序至关重要："狗咬了人"与"人咬了狗"含义截然不同。

因此，我们需要在 Token 嵌入中注入**位置信息**。

### 绝对位置嵌入：加法与拼接的对比

GPT 系列模型采用**绝对位置嵌入**（Absolute Positional Embedding），为每个位置 $$i \in \{0, 1, \dots, L-1\}$$ 学习一个 $$d$$ 维向量：

$$W_p \in \mathbb{R}^{L_{\max} \times d}$$

最终输入嵌入为两者之和：

$$\mathbf{x}_i = \mathbf{x}_i^{\text{token}} + \mathbf{x}_i^{\text{pos}} = W_e[t_i] + W_p[i]$$

一个自然的问题是：**为什么是加法而不是拼接？**

设若采用拼接方式（假设各用 $$d/2$$ 维以保证总维度不变）：

$$\mathbf{x}_i^{\text{concat}} = [W_e'[t_i] \mathbin\Vert W_p'[i]] \in \mathbb{R}^d$$

加法相比于拼接有以下优势：

1. **参数量**：加法只需分别学习 $$v \times d$$ 和 $$L_{\max} \times d$$ 的参数。拼接若要达到相同的表示能力（每个子空间 $$d/2$$ 维），信息容量减半。

2. **交互灵活性**：加法允许 Token 语义和位置信息在相同的 $$d$$ 维空间中进行线性组合。在自注意力机制的 Query-Key 点积中：
   $$q_i \cdot k_j = (W_Q(\mathbf{x}_i^{\text{token}} + \mathbf{x}_i^{\text{pos}})) \cdot (W_K(\mathbf{x}_j^{\text{token}} + \mathbf{x}_j^{\text{pos}}))$$
   展开后会产生交叉项 $$W_Q\mathbf{x}_i^{\text{pos}} \cdot W_K\mathbf{x}_j^{\text{token}}$$，自然地捕捉了"位置 $$i$$ 关注 Token $$j$$"的交互模式。拼接方式下，这种交叉需要更复杂的权重结构才能实现。

3. **经验验证**：原始 Transformer 论文（Vaswani et al., 2017）实验表明，加法和拼接在性能上接近，但加法的参数量和计算效率更优。实际应用中（GPT、BERT）均采用加法。

### 正弦位置编码的推导

原始 Transformer 不使用可学习位置嵌入，而是采用固定的正弦/余弦编码。对于位置 $$pos$$ 和维度索引 $$i$$：

$$
\begin{aligned}
PE(pos, 2i) &= \sin\left(\frac{pos}{10000^{2i/d}}\right) \\
PE(pos, 2i+1) &= \cos\left(\frac{pos}{10000^{2i/d}}\right)
\end{aligned}
$$

其中 $$i = 0, 1, \dots, d/2 - 1$$。

**推导动机与性质**：

1. **相对位置的可表达性**：对于任意固定的偏移量 $$k$$，$$PE(pos+k)$$ 可以表示为 $$PE(pos)$$ 的线性函数。具体地，由三角恒等式：
   $$\sin(pos + k) = \sin(pos)\cos(k) + \cos(pos)\sin(k)$$
   若我们能保证 $$PE(pos+k, 2i)$$ 和 $$PE(pos+k, 2i+1)$$ 可以由 $$PE(pos, 2i)$$ 和 $$PE(pos, 2i+1)$$ 旋转变换得到，则模型可以学习到相对位置关系。

2. **频率的多尺度设计**：$$\omega_i = 10000^{-2i/d}$$ 形成了从 $$1$$ 到 $$1/10000$$ 的几何级数。对于 $$d = 512$$：
   - $$i = 0$$：$$\omega_0 = 1$$，波长 $$2\pi$$，编码高频细节（相邻位置区分）
   - $$i = d/2 - 1$$：$$\omega_{255} \approx 1/10000$$，波长约 $$62831$$，编码长程位置信息

   这种多尺度设计类似于二进制计数——不同维度以不同频率振荡，使得每个位置的编码向量唯一且结构良好。

3. **外推能力**：由于正弦函数有界（$$[-1, 1]$$），位置编码的值不会随位置增长而发散，这在理论上支持对训练中未见过的更长序列进行推理（尽管实际效果有限）。

**GPT 的差异**：OpenAI 的 GPT 模型不使用正弦位置编码，而是使用**可学习的绝对位置嵌入**。其优势在于：
- 直接在训练数据上优化，无须手动设计频率模式。
- 对于预训练数据长度范围内的位置，通常表现优于固定编码。

缺点是缺乏严格的外推能力（无法处理超过训练最大长度的序列）。这一限制是**上下文窗口**概念的核心。

### 上下文窗口与 GPU 显存的权衡

上下文窗口大小 $$L_{\max}$$ 是 LLM 设计中最关键的参数之一。其与 GPU 显存的关系由注意力机制的计算复杂度决定。

自注意力机制的空间复杂度为 $$O(L^2)$$（需存储 $$L \times L$$ 的注意力权重矩阵），时间复杂度为 $$O(L^2 d)$$（对于多头注意力则为 $$O(L^2 d_{\text{model}})$$）。

设 GPU 显存总量为 $$M$$ GB，模型参数占用为 $$P$$ GB，每层激活值存储为 $$A$$ GB。单个样本在 $$n$$ 层 Transformer 中的内存需求近似为：

$$M_{\text{required}} \approx \underbrace{P}_{\text{模型参数}} + \underbrace{n \cdot B \cdot L \cdot d}_{\text{激活值（线性层）}} + \underbrace{n \cdot B \cdot L^2 \cdot h}_{\text{注意力矩阵}}$$

其中 $$B$$ 为批次大小，$$h$$ 为注意力头数。

对于给定的 $$M$$，可以反解出最大支持的序列长度 $$L_{\max}$$（通常由注意力矩阵的 $$O(L^2)$$ 项主导）。这也解释了为什么长上下文 LLM（如 128K Token）需要大量 GPU 显存，并且通常采用 FlashAttention 等 I/O 感知优化来降低显存占用。

在实践中，$$L_{\max}$$ 的选择是模型能力与硬件约束之间的平衡：

| 模型 | 上下文窗口 | 嵌入维度 | 层数 |
|------|-----------|---------|------|
| GPT-2 | 1024 | 768-1600 | 12-48 |
| GPT-3 | 2048 | 12288 | 96 |
| GPT-4 | 8192-128K | 未公开 | 未公开 |

---

## 2.9 本章小结

本章完整覆盖了从原始文本到 LLM 输入的整个数据预处理流水线：

1. **嵌入**将离散词映射为连续向量（$$\mathbb{R}^d$$），嵌入维度是表达能力与计算效率之间的权衡。
2. **分词**将文本拆分为 Token，词级分词简单但无法处理 OOV 问题。
3. **词表**建立 Token 到整数 ID 的双向映射。
4. **特殊 Token**（`<|unk|>`、`<|endoftext|>`）增强了处理未知词和文档边界的能力。
5. **BPE** 通过迭代合并高频字符对构建子词词表，实现了对任意未知词的覆盖，消除了对 `<|unk|>` 的需求。
6. **滑动窗口**从 Token 序列中生成输入-目标对，样本数 $$M = \lfloor (N-W)/S \rfloor + 1$$。
7. **Token 嵌入层**通过查表将 Token ID 转化为可训练的连续向量。
8. **位置嵌入**通过加法注入位置信息，使得原本位置无关的自注意力机制能感知词序。可学习位置嵌入和正弦位置编码各有优劣。

至此，我们已准备好 LLM 的输入嵌入。下一章将进入 LLM 架构的核心——注意力机制。
