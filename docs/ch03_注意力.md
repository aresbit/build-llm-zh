# 第3章: 注意力机制 (Attention Mechanisms)

> 原书第3章 "Coding attention mechanisms" 中文译版，含关键数学推导的完整扩写。

## 3.1 长序列建模问题

在进入大语言模型 (LLM) 核心的自注意力机制之前，我们首先审视不使用注意力机制的架构所面临的根本性困难。

假设我们想构建一个语言翻译模型。如书中图3.3所示，简单的逐词翻译无法处理不同语言之间的语法结构差异。例如德语句子 "Kannst du mir helfen diesen Satz zu uebersetzen" 不能逐词翻译为 "Can you me help this sentence to translate"——正确的翻译 "Can you help me to translate this sentence" 需要将动词 "helfen" 与介词 "zu" 进行跨位置的重新排列。这意味着翻译模型必须能够捕捉词语之间的**长距离依赖关系**。

在 Transformer 出现之前，**编码器-解码器 RNN** (Recurrent Neural Network) 是最流行的机器翻译架构。RNN 的核心特征在于：前一步的输出作为当前步的输入，形成一个循环处理链。对于编码器-解码器 RNN：

- **编码器**逐个处理输入 token，在每个时间步更新其隐藏状态 $$\mathbf{h}_t$$，试图在最终隐藏状态中压缩整个输入句子的含义
- **解码器**从该最终隐藏状态出发，逐 token 生成译文，同样在每个时间步更新自己的隐藏状态

形式化地，RNN 编码器的递推关系为：

$$\mathbf{h}_t = \tanh(\mathbf{W}_{hh} \mathbf{h}_{t-1} + \mathbf{W}_{xh} \mathbf{x}_t + \mathbf{b}_h)$$

其中 $$\mathbf{W}_{hh}$$ 和 $$\mathbf{W}_{xh}$$ 是权重矩阵，$$\mathbf{b}_h$$ 是偏置项，$$\tanh$$ 是激活函数。

RNN 的根本缺陷在于：解码阶段只依赖编码器的最终隐藏状态 $$\mathbf{h}_T$$（对于长度为 $$T$$ 的输入序列），无法**直接访问**编码器中更早的隐藏状态 $$\mathbf{h}_1, \mathbf{h}_2, \ldots, \mathbf{h}_{T-1}$$。对于复杂的长句，跨长距离的依赖关系容易在压缩过程中丢失——这种现象有时被称为"信息瓶颈" (information bottleneck)。这就直接催生了注意力机制的设计动机——让解码器在每一步生成时，能够**有选择地访问输入序列的所有位置**。

## 3.2 用注意力机制捕捉数据依赖

Bahdanau 等人于 2014 年提出的注意力机制（Bahdanau attention，也称为加性注意力 additive attention）修改了编码器-解码器 RNN：解码器可以在每一步生成时，根据当前状态动态地关注输入序列的不同部分，而不仅仅依赖一个固定的最终隐藏状态。

注意力机制的核心思想可形式化为：对于当前正在生成的输出 token，不同的输入 token 具有不同的重要性。这些重要性由**注意力权重**量化——注意力权重越大，表示该输入 token 对当前输出的贡献越大。在 Bahdanau 注意力中，注意力权重通过一个**可训练的对齐模型** $$a(\mathbf{h}_{t-1}^{\text{dec}}, \mathbf{h}_j^{\text{enc}})$$ 计算，该模型以解码器前一步的隐藏状态和编码器的每个隐藏状态为输入，输出一个标量分数。

三年后的 2017 年，Vaswani 等人在划时代论文 "Attention Is All You Need" 中发现 RNN 架构并非必需，提出了完全基于**自注意力机制**的 Transformer 架构。Transformer 中的**自注意力机制**允许输入序列中的每个位置在计算其表示时，同时参考同一序列中所有其他位置的信息。自注意力是当代 GPT 系列等 LLM 最核心的计算单元，也是本章的重点。

"自注意力"中的"自"指的是注意力权重完全在同一序列内部计算——序列中的每个元素关注该序列中的其他所有元素（包括自身），以评估和捕捉序列内部的依赖关系。这与跨序列注意力（如编码器-解码器注意力）形成本质区别：跨序列注意力关注的是两个不同序列之间的关系（如源语言与目标语言），而自注意力关注的是同一序列内部的关系。

在本章中，我们将从零开始逐步实现四种注意力机制变体，每种都在前一种的基础上扩展：

1. **简化的自注意力**（无训练权重）——理解注意力的基本机制
2. **带权重的自注意力**（缩放点积注意力）——引入可训练的 Query/Key/Value 投影
3. **因果注意力**（加掩码）——阻止模型访问未来 token，实现自回归生成
4. **多头注意力**（并行多头）——多个注意力头并行运算，提升表达能力

我们将看到，虽然注意力机制的数学形式和代码实现一开始看起来比较复杂，但只要掌握了核心的矩阵乘法和 softmax 归一化流程，一切都会变得清晰。我们将通过**逐维度追踪**的方式来厘清每一步的输入输出形状。

## 3.3 用自注意力关注输入的不同部分

### 3.3.1 无训练权重的简化自注意力——完整推导

假设我们的输入是一个包含 $$T$$ 个 token 的句子，每个 token 已被转换为一个 $$d$$ 维的嵌入向量（关于嵌入向量的获取，参见第2章）。将整个输入序列表示为矩阵：

$$\mathbf{X} = \begin{bmatrix} \mathbf{x}^{(1)} \\ \mathbf{x}^{(2)} \\ \vdots \\ \mathbf{x}^{(T)} \end{bmatrix} \in \mathbb{R}^{T \times d}$$

其中 $$\mathbf{x}^{(i)} \in \mathbb{R}^{d}$$ 是第 $$i$$ 个 token 的行向量嵌入。

**自注意力的目标**：对于每个输入位置 $$i$$，计算一个**上下文向量** $$\mathbf{z}^{(i)} \in \mathbb{R}^{d}$$，该向量融合了整个序列中所有 token 的信息——不是孤立地表示 token $$i$$，而是表示"在序列上下文中 token $$i$$ 应该如何被理解"。

上下文向量 $$\mathbf{z}^{(i)}$$ 是通过对所有输入嵌入 $$\mathbf{x}^{(j)}$$（$$j = 1, \ldots, T$$）进行**加权求和**得到的。权重 $$\alpha_{ij}$$ 称为注意力权重，表示 token $$j$$ 对 token $$i$$ 的"重要程度"。注意力权重越大，$$\mathbf{x}^{(j)}$$ 对 $$\mathbf{z}^{(i)}$$ 的贡献就越大。

**简化自注意力的三步计算流程：**

**第1步：计算注意力分数 (attention scores)**

以第2个 token $$\mathbf{x}^{(2)}$$（对应句子 "Your journey starts with one step" 中的 "journey"）作为查询，计算它与所有输入 token 的点积，得到注意力分数：

$$\omega_{2j} = \mathbf{x}^{(2)} \cdot \mathbf{x}^{(j)} = \sum_{k=1}^{d} x^{(2)}_k \cdot x^{(j)}_k, \quad j = 1, 2, \ldots, T$$

**点积的几何含义：** 在欧氏空间中，两个向量的点积满足 $$\mathbf{a} \cdot \mathbf{b} = \|\mathbf{a}\| \|\mathbf{b}\| \cos \theta$$，其中 $$\theta$$ 是夹角。因此点积越大，意味着两个向量的方向越一致，对应 token 的语义越相似。在上下文向量计算中，语义更相似的 token 获得更高的注意力权重，这符合直觉——模型应该更关注与当前 token 语义相关的其他 token。

将上述计算扩展到所有 token 对，得到**注意力分数矩阵** $$\boldsymbol{\Omega} \in \mathbb{R}^{T \times T}$$：

$$\Omega_{ij} = \mathbf{x}^{(i)} \cdot \mathbf{x}^{(j)}$$

$$\boldsymbol{\Omega} = \begin{bmatrix} \mathbf{x}^{(1)}\cdot\mathbf{x}^{(1)} & \mathbf{x}^{(1)}\cdot\mathbf{x}^{(2)} & \cdots & \mathbf{x}^{(1)}\cdot\mathbf{x}^{(T)} \\ \mathbf{x}^{(2)}\cdot\mathbf{x}^{(1)} & \mathbf{x}^{(2)}\cdot\mathbf{x}^{(2)} & \cdots & \mathbf{x}^{(2)}\cdot\mathbf{x}^{(T)} \\ \vdots & \vdots & \ddots & \vdots \\ \mathbf{x}^{(T)}\cdot\mathbf{x}^{(1)} & \mathbf{x}^{(T)}\cdot\mathbf{x}^{(2)} & \cdots & \mathbf{x}^{(T)}\cdot\mathbf{x}^{(T)} \end{bmatrix}$$

用矩阵乘法统一表达（一步计算出所有 $$T^2$$ 个点积）：

$$\boldsymbol{\Omega} = \mathbf{X} \mathbf{X}^{\top} \in \mathbb{R}^{T \times T}$$

这是一个对称矩阵（$$\boldsymbol{\Omega} = \boldsymbol{\Omega}^{\top}$$），因为 $$\mathbf{x}^{(i)}\cdot\mathbf{x}^{(j)} = \mathbf{x}^{(j)}\cdot\mathbf{x}^{(i)}$$。对角元 $$\Omega_{ii}$$ 是每个 token 与自身的点积（等于自身范数的平方），通常（但不总是）是每行的最大值。

**第2步：用 softmax 归一化得到注意力权重**

将注意力分数矩阵的每一行用 softmax 函数归一化，使得对每个查询 token $$i$$，注意力权重构成一个**概率分布**（各项非负，且和为 1）：

$$\alpha_{ij} = \frac{\exp(\Omega_{ij})}{\sum_{k=1}^{T} \exp(\Omega_{ik})}$$

矩阵形式：

$$\mathbf{A} = \text{softmax}(\boldsymbol{\Omega}, \text{dim}=-1), \quad A_{ij} = \alpha_{ij}$$

其中 $$\text{dim}=-1$$ 表示沿最后一个维度（列方向）做 softmax，即对每一行独立归一化。归一化保证：

$$\sum_{j=1}^{T} \alpha_{ij} = 1, \quad \forall i \in \{1, \ldots, T\}$$

注意：此时我们尚未引入缩放因子 $$1/\sqrt{d_k}$$，在简化版本中，直接用未缩放的注意力分数做 softmax 归一化。缩放因子的必要性将在 3.4.2 节给出严格的方差分析证明。

**第3步：计算上下文向量**

用注意力权重对输入嵌入进行加权求和，得到每个位置 $$i$$ 的上下文向量：

$$\mathbf{z}^{(i)} = \sum_{j=1}^{T} \alpha_{ij} \ \mathbf{x}^{(j)}$$

矩阵形式（一次性计算所有上下文向量）：

$$\mathbf{Z} = \mathbf{A} \mathbf{X} \in \mathbb{R}^{T \times d}$$

其中 $$\mathbf{Z}$$ 的第 $$i$$ 行 $$\mathbf{z}^{(i)}$$ 是 token $$i$$ 的上下文向量——本质上是所有输入嵌入以注意力权重为系数的**凸组合**，因为权重非负且和为 1。

**总结——简化自注意力的三条核心公式：**

$$\boldsymbol{\Omega} = \mathbf{X} \mathbf{X}^{\top} \quad (T \times T)$$

$$\alpha_{ij} = \frac{\exp(\Omega_{ij})}{\sum_{k} \exp(\Omega_{ik})}$$

$$\mathbf{z}^{(i)} = \sum_{j} \alpha_{ij} \mathbf{x}^{(j)}$$

这三步——(1) 计算相似度分数，(2) 归一化为概率分布，(3) 加权聚合——是所有注意力机制变体共有的核心框架。

**数值实例：** 考虑由6个 token 组成的输入句子 "Your journey starts with one step"，每个 token 的嵌入维度为 $$d=3$$。输入矩阵 $$\mathbf{X} \in \mathbb{R}^{6 \times 3}$$（具体数值见原书）。以第2个 token "journey" 作为查询，计算其与所有6个 token 的点积分数，再经 softmax 归一化得到注意力权重 $$[0.1385, 0.2379, 0.2333, 0.1240, 0.1082, 0.1581]$$。最终的上下文向量 $$\mathbf{z}^{(2)} = \sum_{j=1}^{6} \alpha_{2j} \mathbf{x}^{(j)}$$ 是一个融合了整句信息的3维向量。

### 3.3.2 为所有输入 Token 计算注意力权重

对于包含 $$T$$ 个 token 的输入 $$\mathbf{X}$$，我们可以通过矩阵运算一次性完成上述三步，而不需要显式的 for 循环：

1. **注意力分数矩阵**：$$\boldsymbol{\Omega} = \mathbf{X} \mathbf{X}^{\top}$$，形状 $$(T, T)$$。逐元素计算复杂度为 $$O(T^2 \cdot d)$$
2. **注意力权重矩阵**：$$\mathbf{A} = \text{softmax}(\boldsymbol{\Omega}, \text{dim}=-1)$$，沿每行的列方向做 softmax
3. **上下文向量矩阵**：$$\mathbf{Z} = \mathbf{A} \mathbf{X}$$，形状 $$(T, d)$$

使用矩阵乘法替代显式的 Python for 循环有两个核心优势：(1) 代码更简洁、更不易出错；(2) 可以利用 GPU 的大规模并行计算能力——现代 GPU 上的矩阵乘法经过高度优化，比逐元素的 Python 循环快几个数量级。

## 3.4 带可训练权重的自注意力（缩放点积注意力）

简化版自注意力的根本局限在于**没有任何可训练参数**——模型无法通过学习来优化它对不同 token 之间关系的"理解"。每个 token 对之间的注意力权重完全由固定的嵌入向量决定，模型无法根据训练数据调整哪些关系模式应该被关注。

本节引入三个可训练权重矩阵 $$\mathbf{W}_q$$、$$\mathbf{W}_k$$、$$\mathbf{W}_v$$，将输入投影到**查询**、**键**、**值**空间。这个三元组投影机制是 Transformer 原始论文提出的**缩放点积注意力**的核心。

### 3.4.1 Query、Key、Value 的完整推导

**动机——来自信息检索的类比：** 为什么引入三个不同的投影而不是直接使用输入嵌入？

- **Query** $$\mathbf{q}$$——类比数据库中的搜索查询。它代表当前 token "想要查找什么"——即当前 token 试图从序列中获取什么信息
- **Key** $$\mathbf{k}$$——类比数据库中的索引键。它代表每个 token "可以被匹配的标识"——即该 token 可以用来匹配查询的"标签"
- **Value** $$\mathbf{v}$$——类比键值对中的值。它代表每个 token "实际承载的内容"——即该 token 被检索到时实际提供的信息

这个设计的关键洞察在于：**匹配用的表示**与**被聚合的内容表示**被**解耦**了。同一个 token 可以有不同的"标识"和"内容"。例如，在句子 "The cat sat on the mat" 中，"cat" 这个词用来匹配查询时（作为 key）可能强调它的名词词性，而作为被聚合的内容时（作为 value）可能侧重于它的语义含义。

**形式化定义：**

设有输入序列 $$\mathbf{X} \in \mathbb{R}^{T \times d_{\text{in}}}$$（$$T$$ 个 token，每个嵌入维度为 $$d_{\text{in}}$$）。定义三个可训练权重矩阵：

$$\mathbf{W}_q \in \mathbb{R}^{d_{\text{in}} \times d_k}, \quad \mathbf{W}_k \in \mathbb{R}^{d_{\text{in}} \times d_k}, \quad \mathbf{W}_v \in \mathbb{R}^{d_{\text{in}} \times d_v}$$

通过矩阵乘法将输入嵌入投影到三个不同的表示空间：

$$\mathbf{Q} = \mathbf{X} \mathbf{W}_q \quad (T \times d_k)$$

$$\mathbf{K} = \mathbf{X} \mathbf{W}_k \quad (T \times d_k)$$

$$\mathbf{V} = \mathbf{X} \mathbf{W}_v \quad (T \times d_v)$$

注意 $$\mathbf{W}_q$$ 和 $$\mathbf{W}_k$$ 的输出维度必须相同（都是 $$d_k$$），因为后续要做 query-key 点积，要求维度匹配。$$\mathbf{W}_v$$ 的输出维度 $$d_v$$ 可以与 $$d_k$$ 不同。在实践中，GPT 类模型通常设置 $$d_k = d_v = d_{\text{in}} = d_{\text{out}}$$（即所有维度相等），但为了推导时的清晰性，我们分开标记。

**维度分析表（单样本，无 batch 维度）：**

| 符号 | 含义 | 形状 |
|------|------|------|
| $$\mathbf{X}$$ | 输入嵌入矩阵 | $$T \times d_{\text{in}}$$ |
| $$\mathbf{W}_q$$ | 查询权重矩阵 | $$d_{\text{in}} \times d_k$$ |
| $$\mathbf{W}_k$$ | 键权重矩阵 | $$d_{\text{in}} \times d_k$$ |
| $$\mathbf{W}_v$$ | 值权重矩阵 | $$d_{\text{in}} \times d_v$$ |
| $$\mathbf{Q}$$ | 查询矩阵 | $$T \times d_k$$ |
| $$\mathbf{K}$$ | 键矩阵 | $$T \times d_k$$ |
| $$\mathbf{V}$$ | 值矩阵 | $$T \times d_v$$ |
| $$\mathbf{Q}\mathbf{K}^{\top}$$ | 注意力分数矩阵 | $$T \times T$$ |
| $$\text{softmax}(\mathbf{Q}\mathbf{K}^{\top}/\sqrt{d_k})$$ | 注意力权重矩阵 | $$T \times T$$ |
| $$\text{softmax}(\cdot) \mathbf{V}$$ | 上下文向量矩阵 | $$T \times d_v$$ |

**注意力分数的计算：**

$$\boldsymbol{\Omega} = \mathbf{Q} \mathbf{K}^{\top} = (\mathbf{X} \mathbf{W}_q)(\mathbf{X} \mathbf{W}_k)^{\top} \in \mathbb{R}^{T \times T}$$

展开为标量形式——第 $$i$$ 个查询与第 $$j$$ 个键的点积：

$$\omega_{ij} = \mathbf{q}^{(i)} \cdot \mathbf{k}^{(j)} = \sum_{m=1}^{d_k} q^{(i)}_m \cdot k^{(j)}_m$$

这里 $$\mathbf{q}^{(i)}$$ 和 $$\mathbf{k}^{(j)}$$ 分别是 $$\mathbf{Q}$$ 的第 $$i$$ 行和 $$\mathbf{K}$$ 的第 $$j$$ 行（均为 $$d_k$$ 维行向量）。

### 3.4.2 缩放因子 $$\sqrt{d_k}$$ 的必要性——严格的方差分析

这是本章最关键的数学推导之一，也是 Transformer 论文中一个重要但常被忽视的设计细节。为什么需要在 softmax 之前除以 $$\sqrt{d_k}$$？

**假设条件：** 在模型初始化时（使用如 Xavier/Glorot 初始化），查询向量 $$\mathbf{q}$$ 的每个分量和键向量 $$\mathbf{k}$$ 的每个分量近似独立且服从标准正态分布 $$\mathcal{N}(0, 1)$$。

> **注：** 严格意义上，$$\mathbf{q} = \mathbf{x} \mathbf{W}_q$$ 的精确分布由输入 $$\mathbf{x}$$ 和权重 $$\mathbf{W}_q$$ 的分布联合决定。在适当的初始化策略下（如 $$\mathbf{W}_q$$ 各元素服从 $$\mathcal{N}(0, 1/d_{\text{in}})$$），$$\mathbf{q}$$ 的各分量近似 $$\mathcal{N}(0, 1)$$。为简化推导，我们假设各分量独立且服从 $$\mathcal{N}(0, 1)$$。

**推导点积的方差：**

考虑单个点积 $$\mathbf{q} \cdot \mathbf{k} = \sum_{i=1}^{d_k} q_i k_i$$。

**步骤1：单分量的期望和方差。** 由于 $$q_i$$ 和 $$k_i$$ 独立，且 $$\mathbb{E}[q_i] = \mathbb{E}[k_i] = 0$$：

$$\mathbb{E}[q_i k_i] = \mathbb{E}[q_i] \cdot \mathbb{E}[k_i] = 0 \cdot 0 = 0$$

**步骤2：单分量乘积的方差。**

$$\text{Var}(q_i k_i) = \mathbb{E}[(q_i k_i)^2] - (\mathbb{E}[q_i k_i])^2$$

由于 $$q_i$$ 和 $$k_i$$ 独立，$$\mathbb{E}[(q_i k_i)^2] = \mathbb{E}[q_i^2] \cdot \mathbb{E}[k_i^2]$$。因为 $$q_i, k_i \sim \mathcal{N}(0, 1)$$，有 $$\mathbb{E}[q_i^2] = \text{Var}(q_i) = 1$$，同理 $$\mathbb{E}[k_i^2] = 1$$。因此：

$$\mathbb{E}[(q_i k_i)^2] = 1 \cdot 1 = 1$$

$$\text{Var}(q_i k_i) = 1 - 0 = 1$$

**步骤3：点积的总方差。** 因为 $$d_k$$ 个不同的分量对之间相互独立，和的方差等于方差之和：

$$\text{Var}(\mathbf{q} \cdot \mathbf{k}) = \text{Var}\left(\sum_{i=1}^{d_k} q_i k_i\right) = \sum_{i=1}^{d_k} \text{Var}(q_i k_i) = \sum_{i=1}^{d_k} 1 = d_k$$

**关键结论：** 点积的方差为 $$d_k$$，标准差为 $$\sqrt{d_k}$$。

**为什么这会导致问题？** 对于 GPT-2 的中等配置（$$d_k = 768$$），点积的标准差约为 $$\sqrt{768} \approx 27.7$$。这意味着某些 $$\omega_{ij}$$ 的绝对值可能处于 $$\pm 80$$ 甚至更大的范围（根据正态分布的 $$3\sigma$$ 法则）。

考察 softmax 函数在极端输入下的行为：

$$\text{softmax}(\boldsymbol{\omega})_i = \frac{e^{\omega_i}}{\sum_j e^{\omega_j}}$$

当某个 $$\omega_i$$ 远大于其余分量时，$$e^{\omega_i}$$ 将在分子和分母中同时主导，导致：

$$\lim_{\omega_i - \max_j \omega_j \to \infty} \text{softmax}(\boldsymbol{\omega})_i = 1, \quad \text{其余 } \to 0$$

即注意力分布退化为接近 one-hot 向量。在这种情况下，softmax 的梯度趋近于零（在 3.4.3 节将严格证明），严重阻碍训练——这就是**梯度消失**问题。

**解决方案——缩放：**

将每个注意力分数除以 $$\sqrt{d_k}$$：

$$\text{Attention}(\mathbf{Q}, \mathbf{K}, \mathbf{V}) = \text{softmax}\left(\frac{\mathbf{Q} \mathbf{K}^{\top}}{\sqrt{d_k}}\right) \mathbf{V}$$

缩放后的点积方差变为：

$$\text{Var}\left(\frac{\mathbf{q} \cdot \mathbf{k}}{\sqrt{d_k}}\right) = \frac{\text{Var}(\mathbf{q} \cdot \mathbf{k})}{d_k} = \frac{d_k}{d_k} = 1$$

方差被归一化为 1，softmax 的输入保持在合理范围内（绝大多数值在 $$[-3, 3]$$ 之间），避免了梯度过小的问题。这就是**缩放点积注意力**名称的由来。

### 3.4.3 Softmax 函数的梯度推导——雅可比矩阵完整推导

为完整理解训练过程中的反向传播（特别是梯度如何通过注意力模块流动），我们严格推导 softmax 函数的雅可比矩阵。

**定义：** 对于输入向量 $$\mathbf{z} = [z_1, z_2, \ldots, z_n]^{\top}$$，softmax 函数的第 $$i$$ 个输出分量定义为：

$$\text{softmax}_i(\mathbf{z}) = p_i = \frac{e^{z_i}}{\sum_{k=1}^{n} e^{z_k}}$$

记分母（配分函数）为 $$S = \sum_{k=1}^{n} e^{z_k}$$，则 $$p_i = e^{z_i} / S$$。

**求导——使用商法则分两种情形：**

**情形 1：对角线元素 ($$i = j$$)**

$$\frac{\partial p_i}{\partial z_i} = \frac{\partial}{\partial z_i} \left( \frac{e^{z_i}}{S} \right)$$

应用商法则 $$\frac{\partial}{\partial z_i}\left(\frac{u}{v}\right) = \frac{u'v - uv'}{v^2}$$，其中 $$u = e^{z_i}$$，$$v = S$$，$$u' = e^{z_i}$$，$$v' = e^{z_i}$$（因为对 $$S$$ 关于 $$z_i$$ 求导仅影响 $$e^{z_i}$$ 项）：

$$\frac{\partial p_i}{\partial z_i} = \frac{e^{z_i} \cdot S - e^{z_i} \cdot e^{z_i}}{S^2} = \frac{e^{z_i}}{S} - \frac{(e^{z_i})^2}{S^2} = p_i - p_i^2 = p_i(1 - p_i)$$

**情形 2：非对角线元素 ($$i \neq j$$)**

$$\frac{\partial p_i}{\partial z_j} = \frac{\partial}{\partial z_j} \left( \frac{e^{z_i}}{S} \right)$$

此时 $$u = e^{z_i}$$（与 $$z_j$$ 无关，$$u' = 0$$），$$v = S$$，$$v' = e^{z_j}$$：

$$\frac{\partial p_i}{\partial z_j} = \frac{0 \cdot S - e^{z_i} \cdot e^{z_j}}{S^2} = -\frac{e^{z_i}}{S} \cdot \frac{e^{z_j}}{S} = -p_i p_j$$

**统一形式（利用 Kronecker delta $$\delta_{ij}$$）：**

$$\frac{\partial p_i}{\partial z_j} = p_i (\delta_{ij} - p_j)$$

其中 $$\delta_{ij} = \begin{cases} 1 & \text{if } i = j \\ 0 & \text{if } i \neq j \end{cases}$$

**雅可比矩阵的显式形式：**

将上述导数排列为一个 $$n \times n$$ 矩阵：

$$\mathbf{J}_{\text{softmax}} = \text{diag}(\mathbf{p}) - \mathbf{p} \mathbf{p}^{\top}$$

展开为：

$$\mathbf{J}_{\text{softmax}} = \begin{bmatrix} p_1(1-p_1) & -p_1 p_2 & \cdots & -p_1 p_n \\ -p_2 p_1 & p_2(1-p_2) & \cdots & -p_2 p_n \\ \vdots & \vdots & \ddots & \vdots \\ -p_n p_1 & -p_n p_2 & \cdots & p_n(1-p_n) \end{bmatrix}$$

**雅可比矩阵的两个重要性质：**

1. **梯度消失条件：** 当 $$\mathbf{p}$$ 趋近于 one-hot 向量时（如 $$p_i \to 1$$ 且 $$p_j \to 0$$ for $$j \neq i$$），所有导数趋近于 0（因为 $$p_k(1-p_k) \to 0$$ 且 $$p_i p_j \to 0$$）。此时梯度无法有效回传——这就是 3.4.2 节中缩放因子 $$\sqrt{d_k}$$ 要防止的情况。通过将点积方差控制在 1 左右，softmax 的输入不至于过大，输出分布保持适度平滑，从而维持有效的梯度流。

2. **与交叉熵损失的优雅组合：** 在分类任务中，softmax 后接交叉熵损失 $$\mathcal{L} = -\sum_i y_i \log p_i$$ 时，利用链式法则可以证明 $$\frac{\partial \mathcal{L}}{\partial z_i} = p_i - y_i$$。这个极简洁的梯度形式是 softmax + 交叉熵组合被广泛使用的重要原因之一。

**Softmax 的数值稳定性——Log-Sum-Exp 技巧：**

在实际的数值计算中，直接计算 $$e^{z_i}$$ 可能因 $$z_i$$ 过大而导致上溢 (overflow, 超出浮点数表示范围) 或因 $$z_i$$ 过小（绝对值很大的负数）而导致下溢 (underflow)。标准的数值稳定实现是**Log-Sum-Exp 技巧**：

$$\text{softmax}(z_i) = \frac{\exp(z_i - \max_k z_k)}{\sum_{j} \exp(z_j - \max_k z_k)}$$

可以证明这与原定义**数学上严格等价**，因为分子分母同时乘以 $$e^{-\max_k z_k}$$：

$$\frac{\exp(z_i - m)}{\sum_j \exp(z_j - m)} = \frac{e^{z_i} / e^{m}}{\sum_j e^{z_j} / e^{m}} = \frac{e^{z_i}}{\sum_j e^{z_j}}$$

其中 $$m = \max_k z_k$$。减去最大值后，$$\exp$$ 的参数最大为 0，最小为一个有限的负数（最大值减去其余值），完全避免了上溢问题。PyTorch 的 `torch.softmax` 内部正是用此技巧实现的。

**缩放点积注意力与加性注意力的对比：**

Transformer 原始论文选择了点积注意力（而非 Bahdanau 的加性注意力），主要理由有二：(1) 点积注意力可以通过高度优化的矩阵乘法实现，在 GPU 上计算效率极高；(2) 引入缩放因子 $$\sqrt{d_k}$$ 后，其效果与加性注意力相当。加性注意力的计算形式为 $$a(\mathbf{q}, \mathbf{k}) = \mathbf{v}^{\top} \tanh(\mathbf{W}_1 \mathbf{q} + \mathbf{W}_2 \mathbf{k})$$（需要额外的可训练参数 $$\mathbf{v}, \mathbf{W}_1, \mathbf{W}_2$$），虽然理论上更具表达力，但在实践中开销更大且增益有限。

### 3.4.4 完整的带权自重注意力机制

将 Query、Key、Value 投影、缩放因子，以及 softmax 归一化组合在一起，得到完整的缩放点积注意力公式：

$$\mathbf{Q} = \mathbf{X} \mathbf{W}_q, \quad \mathbf{K} = \mathbf{X} \mathbf{W}_k, \quad \mathbf{V} = \mathbf{X} \mathbf{W}_v$$

$$\mathbf{Z} = \text{Attention}(\mathbf{Q}, \mathbf{K}, \mathbf{V}) = \text{softmax}\left(\frac{\mathbf{Q} \mathbf{K}^{\top}}{\sqrt{d_k}}\right) \mathbf{V}$$

展开为逐 token 的标量形式（第 $$i$$ 个 token 的上下文向量）：

$$\mathbf{z}^{(i)} = \sum_{j=1}^{T} \alpha_{ij} \mathbf{v}^{(j)}$$

其中注意力权重为：

$$\alpha_{ij} = \frac{\exp(\omega_{ij} / \sqrt{d_k})}{\sum_{k=1}^{T} \exp(\omega_{ik} / \sqrt{d_k})}, \quad \omega_{ij} = \mathbf{q}^{(i)} \cdot \mathbf{k}^{(j)}$$

**关于 $$\mathbf{W}_v$$（值权重矩阵）的维度：** 值投影的输出维度 $$d_v$$ 决定了上下文向量的维度。在 GPT 类模型中，通常设置 $$d_v = d_{\text{out}}$$（模型的隐藏维度），使得自注意力模块的输入和输出维度相同——这是构建深层网络所必需的，下一章将看到这如何与残差连接配合。

**代码实现要点——为什么推荐 `nn.Linear`：** 在 PyTorch 中，使用 `nn.Linear(d_in, d_out, bias=False)` 来实现 $$\mathbf{X}\mathbf{W}$$ 的线性投影，而不是手动定义 `nn.Parameter(torch.rand(d_in, d_out))`。`nn.Linear` 内部使用了 Kaiming 均匀初始化，有效地控制了前向传播中激活值的方差，避免了深层网络中的梯度消失或爆炸问题。此外，`nn.Linear` 在内部将权重矩阵以**转置形式**存储（形状 $$(d_{\text{out}}, d_{\text{in}})$$ 而不是 $$(d_{\text{in}}, d_{\text{out}})$$），以优化矩阵乘法的内存访问模式。这就是为什么在 Exercise 3.1 中，将 `SelfAttention_v2` 的权重赋值给 `SelfAttention_v1` 需要进行转置操作。

## 3.5 用因果注意力隐藏未来 Token

### 3.5.1 因果掩码的推导

在 GPT 类**自回归**语言模型中，文本生成是从左到右逐个 token 进行的——在预测位置 $$i$$ 的输出时，模型只能看到位置 $$1, 2, \ldots, i$$ 的 token。如果在训练时允许模型"偷看"位置 $$i+1$$ 及之后的 token，就会发生**信息泄露**。这意味着训练时的输入信息量大于推理时（推理时确实看不到未来），导致训练-推理不一致，模型在推理时表现显著下降。

**因果注意力**，也称**掩码注意力**，通过在注意力分数矩阵上施加一个上三角掩码来强制实现从左到右的约束。

**定义因果掩码矩阵 $$\mathbf{M} \in \mathbb{R}^{T \times T}$$：**

$$M_{ij} = \begin{cases} 0 & \text{if } j \leq i \quad \text{(允许关注——当前及之前的 token)} \\ -\infty & \text{if } j > i \quad \text{(禁止关注——未来的 token)} \end{cases}$$

显式矩阵形式（对于 $$T=6$$）：

$$\mathbf{M} = \begin{bmatrix} 0 & -\infty & -\infty & -\infty & -\infty & -\infty \\ 0 & 0 & -\infty & -\infty & -\infty & -\infty \\ 0 & 0 & 0 & -\infty & -\infty & -\infty \\ 0 & 0 & 0 & 0 & -\infty & -\infty \\ 0 & 0 & 0 & 0 & 0 & -\infty \\ 0 & 0 & 0 & 0 & 0 & 0 \end{bmatrix}$$

**编码实现：** 在 PyTorch 中，$$\mathbf{M}$$ 通过两步生成：先用 `torch.triu(torch.ones(T, T), diagonal=1)` 生成上三角（不含对角线）为 1、其余为 0 的矩阵，再通过 `masked_fill(mask.bool(), -torch.inf)` 将 1 替换为 $$-\infty$$。最终的因果掩码通过 `register_buffer` 注册，随模型自动在 CPU/GPU 间迁移。

**将掩码整合到注意力计算中：**

$$\mathbf{Z} = \text{softmax}\left(\frac{\mathbf{Q} \mathbf{K}^{\top}}{\sqrt{d_k}} + \mathbf{M}\right) \mathbf{V}$$

关键在于理解为什么在 softmax **之前**加上掩码（而不是之后）：

**数学原理——为什么 $$-\infty$$ 通过 softmax 后精确变为 0：**

对于第 $$i$$ 行，当 $$j > i$$ 时，$$M_{ij} = -\infty$$。该位置进入 softmax 的指数运算：

$$\exp\left(\frac{\omega_{ij}}{\sqrt{d_k}} + M_{ij}\right) = \exp\left(\frac{\omega_{ij}}{\sqrt{d_k}} + (-\infty)\right)$$

由于 $$\omega_{ij} / \sqrt{d_k}$$ 是有限实数：

$$\exp(\text{有限实数} - \infty) = \exp(-\infty) = 0$$

因此 softmax 后，位置 $$j > i$$ 的注意力权重精确为 0：

$$\alpha_{ij} = 0 \quad \text{for all } j > i$$

这确保了在计算 token $$i$$ 的上下文向量时，未来 token ($$j > i$$) 的**贡献精确为零**，信息不会从未来泄露到当前位置。

**掩码的演化过程（以 $$T=4$$ 为例）：**

掩码前的注意力分数：

$$\begin{bmatrix} \omega_{11} & \omega_{12} & \omega_{13} & \omega_{14} \\ \omega_{21} & \omega_{22} & \omega_{23} & \omega_{24} \\ \omega_{31} & \omega_{32} & \omega_{33} & \omega_{34} \\ \omega_{41} & \omega_{42} & \omega_{43} & \omega_{44} \end{bmatrix}$$

叠加掩码 $$\mathbf{M}$$ 后（$$+\mathbf{M}$$）：

$$\begin{bmatrix} \omega_{11} & -\infty & -\infty & -\infty \\ \omega_{21} & \omega_{22} & -\infty & -\infty \\ \omega_{31} & \omega_{32} & \omega_{33} & -\infty \\ \omega_{41} & \omega_{42} & \omega_{43} & \omega_{44} \end{bmatrix}$$

softmax 归一化后（下三角矩阵，各行独立归一化）：

$$\begin{bmatrix} 1.0000 & 0 & 0 & 0 \\ \alpha_{21} & \alpha_{22} & 0 & 0 \\ \alpha_{31} & \alpha_{32} & \alpha_{33} & 0 \\ \alpha_{41} & \alpha_{42} & \alpha_{43} & \alpha_{44} \end{bmatrix}$$

例如在第一行，仅 $$\omega_{11}$$ 为非无穷值，softmax 后 $$\alpha_{11} = 1.0$$（其余为 0）。第二行的 $$\alpha_{21}$$ 和 $$\alpha_{22}$$ 满足 $$\alpha_{21} + \alpha_{22} = 1$$，且 $$\alpha_{22} > \alpha_{21}$$（因为对角元 $$\omega_{22}$$ 通常较大）。

**另一种实现路径的对比——先 softmax 再掩码：** 书中展示了一种替代方案：先对所有分数做 softmax 得到注意力权重，再用 0-1 掩码将未来位置的权重置零，最后对每行重新归一化。数学上，这两种方法是**等价的**——先 softmax 再掩码+重归一化的结果与直接在 softmax 前加 $$-\infty$$ 掩码完全相同。但后一种方法（先加 $$-\infty$$ 再 softmax）只需要一次 softmax 调用，计算效率更高，且概念上更优雅。

我们可以严格证明这两种方式的等价性。设未掩码的 softmax 权重为 $$\alpha_{ij} = e^{\omega_{ij}} / \sum_{k=1}^{T} e^{\omega_{ik}}$$。若在 softmax 后将 $$j > i$$ 位置的权重置零并重新归一化，则得到：

$$\alpha'_{ij} = \begin{cases} \displaystyle\frac{\alpha_{ij}}{\sum_{k=1}^{i} \alpha_{ik}} = \frac{e^{\omega_{ij}}}{\sum_{k=1}^{i} e^{\omega_{ik}}}, & j \leq i \\[12pt] 0, & j > i \end{cases}$$

另一方面，若在 softmax 前加上 $$-\infty$$ 掩码，直接得到：

$$\alpha''_{ij} = \frac{e^{\omega_{ij} + M_{ij}}}{\sum_{k=1}^{T} e^{\omega_{ik} + M_{ik}}} = \frac{e^{\omega_{ij}}}{\sum_{k=1}^{i} e^{\omega_{ik}}}, \quad j \leq i$$

其中 $$e^{-\infty} = 0$$ 使得 $$k > i$$ 项不参与分母求和。显然 $$\alpha'_{ij} = \alpha''_{ij}$$ for all $$i, j$$，两种方法完全等价。先加 $$-\infty$$ 掩码再 softmax 是实践中的标准做法。

### 3.5.2 Dropout 掩码——防止过拟合的正则化手段

深度神经网络极易在训练数据上**过拟合**——模型"记住"了训练样本的细节而非学习可泛化的模式。**Dropout** 是经典的神经网络正则化技术：在每次训练迭代中，以概率 $$p$$ **随机丢弃**（置零）一部分激活值，迫使网络不依赖任何单一的强特征。

在 Transformer 的注意力机制中，dropout 通常应用于两个位置之一：

1. **注意力权重上**（在 softmax 之后）：以概率 $$p$$ 随机将注意力权重置零
2. **注意力输出上**（在加权值向量之后）：以概率 $$p$$ 随机将上下文向量的某些分量置零

本章采用第一种方式（对注意力权重施加 dropout），这是更常见的实践。

**形式化定义：** 对于第 $$i$$ 行的注意力权重向量 $$\boldsymbol{\alpha}_i = [\alpha_{i1}, \alpha_{i2}, \ldots, \alpha_{iT}]$$，施加 dropout 后的输出为：

$$\tilde{\boldsymbol{\alpha}}_i = \frac{1}{1-p} \cdot \mathbf{d} \odot \boldsymbol{\alpha}_i$$

其中 $$\mathbf{d} = [d_1, d_2, \ldots, d_T]$$ 的每个分量独立服从 Bernoulli 分布：

$$d_j \sim \text{Bernoulli}(1-p), \quad P(d_j = 1) = 1-p, \quad P(d_j = 0) = p$$

$$\odot$$ 表示逐元素乘法 (Hadamard 乘积)。

**缩放因子 $$1/(1-p)$$ 的必要性：** 在推理阶段（测试阶段），dropout 被**禁用**——所有注意力权重保持原样。为确保训练和推理时注意力权重的**期望值一致**，训练时需要将保留的权重放大 $$1/(1-p)$$ 倍：

$$\mathbb{E}[\tilde{\boldsymbol{\alpha}}_i] = \frac{1}{1-p} \cdot \mathbb{E}[\mathbf{d}] \odot \boldsymbol{\alpha}_i = \frac{1}{1-p} \cdot (1-p) \cdot \boldsymbol{\alpha}_i = \boldsymbol{\alpha}_i$$

不进行此缩放的后果是：训练时有效注意力权重的期望为 $$(1-p)\boldsymbol{\alpha}_i$$（因为平均只有 $$1-p$$ 比例的权重被保留），推理时则为 $$\boldsymbol{\alpha}_i$$（全部保留）。这种不一致会降低模型在测试集上的表现。缩放因子恰好抵消了丢弃的影响，使得训练和推理时的期望激活值保持一致。

**实际参数：** 在训练 GPT 模型时，通常使用较低的 dropout 率，如 $$p = 0.1$$ 或 $$p = 0.2$$（即保留 90% 或 80% 的权重）。过高的 dropout 率会丢弃太多信息，阻碍模型学习有意义的注意力模式。

### 3.5.3 `register_buffer` 的作用

在 `CausalAttention` 类的实现中，因果掩码通过 `self.register_buffer('mask', ...)` 注册为模型的缓冲区而非参数。`register_buffer` 是 PyTorch 中的关键机制：

- **缓冲区不是可训练参数**：在调用 `model.parameters()` 时，缓冲区不被包含在内，因此优化器不会更新它
- **随模型自动迁移**：当模型通过 `model.to(device)` 移动到 GPU 时，缓冲区自动跟随迁移，避免设备不匹配错误
- **随模型保存/加载**：缓冲区会随 `model.state_dict()` 一起被保存和加载

由于因果掩码在整个训练过程中保持不变（它是一个固定的上三角 $$-\infty$$ 矩阵），将其注册为缓冲区是最合适的选择。

## 3.6 多头注意力 (Multi-Head Attention)

### 3.6.1 核心思想与设计动机

单头自注意力只能捕获输入的一种"关系模式"——它只有一组 $$\mathbf{W}_q, \mathbf{W}_k, \mathbf{W}_v$$ 权重来学习 token 之间的依赖关系。**多头注意力**通过并行运行多个独立的注意力"头"，每个头拥有自己的权重矩阵，从而允许模型同时从多个不同的**表示子空间**关注输入的不同方面。

举个例子：在处理句子 "The cat sat on the mat because it was tired" 时，一个头可能聚焦于**句法关系**（"it" 指代 "cat"），另一个头可能聚焦于**语义关系**（"tired" 与 "sat" 之间的因果关联），再一个头可能关注**位置关系**（相邻词之间的局部依赖）。每个头学习互补的模式，最终的拼接输出融合了所有这些视角的信息。

### 3.6.2 数学推导

设有 $$h$$ 个注意力头。对每个头 $$i \in \{1, 2, \ldots, h\}$$，定义**独立的**可训练权重矩阵：

$$\mathbf{W}_{q}^{(i)} \in \mathbb{R}^{d_{\text{in}} \times d_k}, \quad \mathbf{W}_{k}^{(i)} \in \mathbb{R}^{d_{\text{in}} \times d_k}, \quad \mathbf{W}_{v}^{(i)} \in \mathbb{R}^{d_{\text{in}} \times d_v}$$

每个头独立执行完整的缩放点积注意力计算：

$$\text{head}_i = \text{Attention}\left(\mathbf{X} \mathbf{W}_{q}^{(i)},\ \mathbf{X} \mathbf{W}_{k}^{(i)},\ \mathbf{X} \mathbf{W}_{v}^{(i)}\right)$$

$$= \text{softmax}\left(\frac{(\mathbf{X} \mathbf{W}_{q}^{(i)})(\mathbf{X} \mathbf{W}_{k}^{(i)})^{\top}}{\sqrt{d_k}}\right) (\mathbf{X} \mathbf{W}_{v}^{(i)})$$

其中 $$\text{head}_i \in \mathbb{R}^{T \times d_v}$$。注意每个头内部的计算是**完全独立**的（除了共享输入 $$\mathbf{X}$$），因此不同的头可以学到互补的注意力模式。

将 $$h$$ 个头的输出沿最后一个维度**拼接**：

$$\text{Concat}(\text{head}_1, \text{head}_2, \ldots, \text{head}_h) \in \mathbb{R}^{T \times (h \cdot d_v)}$$

拼接后的矩阵每一行是一个 $$h \cdot d_v$$ 维的向量——它是各头输出在对应位置上的简单并列。最后通过一个**输出投影矩阵** $$\mathbf{W}_O \in \mathbb{R}^{(h \cdot d_v) \times d_{\text{out}}}$$ 将拼接后的表示线性映射到目标维度：

$$\text{MultiHead}(\mathbf{X}) = \text{Concat}(\text{head}_1, \ldots, \text{head}_h) \ \mathbf{W}_O \in \mathbb{R}^{T \times d_{\text{out}}}$$

**关于 $$\mathbf{W}_O$$ 的讨论：** 输出投影矩阵 $$\mathbf{W}_O$$ 允许模型学习如何**融合**来自不同头的信息——而不只是将它们简单地串联在一起。虽然从理论上说这个投影不是严格必需的（因为后续层可以直接处理拼接后的表示），但在实践中 $$\mathbf{W}_O$$ 几乎总是被包含在多头注意力中，它提供了额外的表达能力，使模型能够对来自不同表示子空间的信息进行加权组合。

**多头注意力的总参数量：**

$$\text{参数总数} = h \cdot (d_{\text{in}} \cdot d_k + d_{\text{in}} \cdot d_k + d_{\text{in}} \cdot d_v) + (h \cdot d_v) \cdot d_{\text{out}}$$

$$= h \cdot d_{\text{in}} \cdot (2d_k + d_v) + h \cdot d_v \cdot d_{\text{out}}$$

在 GPT 类模型中，通常设置 $$d_k = d_v = d_{\text{out}} / h$$，此时 $$d_{\text{in}} = d_{\text{out}} = d_{\text{model}}$$，参数量简化为 $$4 \cdot d_{\text{model}}^2$$（**独立于头数 $$h$$**），这是因为：

$$h \cdot d_{\text{in}} \cdot (3 \cdot d_{\text{model}}/h) + h \cdot (d_{\text{model}}/h) \cdot d_{\text{model}} = 3 \cdot d_{\text{model}}^2 + d_{\text{model}}^2 = 4 \cdot d_{\text{model}}^2$$

这个性质意味着**增加注意力头的数量不会增加模型的总参数量**（只要保持总输出维度不变），但会增加计算的细粒度。

### 3.6.3 高效的批量实现——权重拆分法

朴素的实现方式（如 `MultiHeadAttentionWrapper` 类所示）需要在 for 循环中为每个头分别执行 $$\mathbf{X}\mathbf{W}_q^{(i)}$$ 等矩阵乘法，计算开销为 $$h$$ 倍的线性投影。更高效的做法借鉴了**批处理**的思想：**使用一个大的权重矩阵一次性完成所有头的线性投影，再通过 reshape 和 transpose 操作将结果拆分为多个头**。

**关键技巧：** 令总投影维度 $$d_{\text{out}} = h \cdot d_k$$。对 Query 使用一个统一的权重矩阵：

$$\mathbf{W}_q \in \mathbb{R}^{d_{\text{in}} \times d_{\text{out}}}$$

一次矩阵乘法得到：

$$\mathbf{Q} = \mathbf{X} \mathbf{W}_q \in \mathbb{R}^{T \times d_{\text{out}}}$$

这里 $$\mathbf{W}_q$$ 实际上等价于将所有 $$h$$ 个头的 $$\mathbf{W}_q^{(i)}$$ 沿列方向拼接在一起：

$$\mathbf{W}_q = [\mathbf{W}_q^{(1)} \ \vert \ \mathbf{W}_q^{(2)} \ \vert \ \cdots \ \vert \ \mathbf{W}_q^{(h)}]$$

然后通过 reshape 将 $$\mathbf{Q}$$ 从 $$(T, d_{\text{out}})$$ 转换为 $$(T, h, d_k)$$，再 transpose 为 $$(h, T, d_k)$$，使注意力的计算可以批量完成。

**引入批维度 $$B$$ 后的完整维度流（10步追踪）：**

下表以处理一个 mini-batch 的数据为场景（$$B$$ 是 batch 大小，$$T$$ 是序列 token 数，$$d_{\text{in}}$$ 是输入嵌入维度，$$h$$ 是头数，$$d_k$$ 是每头的键/查询维度，$$d_{\text{out}} = h \cdot d_k$$ 是总输出维度）。

| 步骤 | 操作 | 输出形状 | 说明 |
|------|------|----------|------|
| 0 | 输入 $$\mathbf{X}$$ | $$(B, T, d_{\text{in}})$$ | mini-batch 的嵌入输入 |
| 1 | $$\mathbf{Q} = \mathbf{X}\mathbf{W}_q$$ | $$(B, T, d_{\text{out}})$$ | 统一的线性投影 |
| 2 | `Q.view(B, T, h, d_k)` | $$(B, T, h, d_k)$$ | 拆分为 $$h$$ 个头 |
| 3 | `Q.transpose(1, 2)` | $$(B, h, T, d_k)$$ | 将头维度移至第二维 |
| 4 | $$\mathbf{Q}\mathbf{K}^{\top}$$ | $$(B, h, T, T)$$ | 每个头独立的注意力分数 |
| 5 | 缩放 + softmax + mask | $$(B, h, T, T)$$ | 逐头归一化+掩码 |
| 6 | $$\times \mathbf{V}$$ | $$(B, h, T, d_k)$$ | 加权聚合值向量 |
| 7 | `.transpose(1, 2)` | $$(B, T, h, d_k)$$ | 恢复维度顺序 |
| 8 | `.contiguous().view(B, T, d_out)` | $$(B, T, d_{\text{out}})$$ | 展平拼接各头 |
| 9 | $$\times \mathbf{W}_O$$ | $$(B, T, d_{\text{out}})$$ | 输出投影 |
| 10 | 最终输出 | $$(B, T, d_{\text{out}})$$ | 多头注意力结果 |

**为什么需要 `.contiguous()`：** 在 PyTorch 中，`transpose` 操作不改变底层内存布局——它只修改了张量的步长元数据。对非连续张量调用 `.view()` 会报错，因此需要先通过 `.contiguous()` 将张量在内存中重新排列为连续布局。这是 PyTorch 实现中的一个技术细节，但若不处理会导致难以调试的运行时错误。

**批处理矩阵乘法的原理：** PyTorch 的四维张量矩阵乘法（`@` 运算符）自动在最后两个维度上执行标准矩阵乘法，其余的前导维度被视为批处理维度。即，对于形状为 $$(B, h, T, d_k)$$ 的 $$\mathbf{Q}$$ 和形状为 $$(B, h, d_k, T)$$ 的 $$\mathbf{K}^{\top}$$，`Q @ K.transpose(2, 3)` 等价于对每个 batch 的每个头独立执行 $$(T, d_k) \times (d_k, T)$$ 的矩阵乘法，无需任何显式循环。这种零开销的批处理能力是现代深度学习框架的核心竞争力之一。

### 3.6.4 两种实现策略的对比

| 方面 | MultiHeadAttentionWrapper | MultiHeadAttention (权重拆分) |
|------|--------------------------|-------------------------------|
| 实现方式 | 多个独立 `CausalAttention` 实例 | 统一权重矩阵 + reshape/transpose |
| 矩阵乘法次数 | $$h$$ 次 $$\mathbf{X}\mathbf{W}$$ | 1 次（更大的）$$\mathbf{X}\mathbf{W}$$ |
| 内存效率 | 多个独立副本的中间结果 | 单次中间结果，共享内存 |
| GPU 利用率 | 较低（序列执行） | 较高（批量并行） |
| 代码复杂度 | 简单直观 | 涉及 reshape/transpose |
| 灵活性 | 容易为不同头配置不同参数 | 所有头共享相同维度 |

在大规模模型中（如 GPT-2 有 12 头，GPT-3 有 96 头），权重拆分法的效率优势极为显著，是所有实际部署中采用的标准方案。

## 3.7 计算复杂度分析

**单头自注意力的复杂度分解：**

对于输入 $$\mathbf{X} \in \mathbb{R}^{T \times d}$$（假设所有维度统一为 $$d$$ 以便分析）：

1. **线性投影** $$\mathbf{Q}, \mathbf{K}, \mathbf{V} = \mathbf{X}\mathbf{W}$$：每次投影为 $$T$$ 个 token 各做一次 $$(d \times d)$$ 矩阵乘法，复杂度共 $$O(T \cdot d^2)$$

2. **计算注意力分数** $$\mathbf{Q}\mathbf{K}^{\top}$$：$$(T \times d) \cdot (d \times T) = (T \times T)$$ 矩阵乘法，复杂度 $$O(T^2 \cdot d)$$

3. **softmax 归一化**：对 $$T \times T$$ 矩阵每行做 softmax，复杂度 $$O(T^2)$$

4. **加权值** $$\mathbf{A}\mathbf{V}$$：$$(T \times T) \cdot (T \times d) = (T \times d)$$ 矩阵乘法，复杂度 $$O(T^2 \cdot d)$$

**总复杂度：**

$$O(T \cdot d^2 + T^2 \cdot d)$$

对于多头注意力（$$h$$ 个头，每头维度 $$d_k = d/h$$）：

$$O(T \cdot d^2 + h \cdot T^2 \cdot d_k) = O(T \cdot d^2 + T^2 \cdot d)$$

令人瞩目的是，复杂度中 $$h$$ 被**消去**——因为虽然头数变多，但每头的维度等比例缩小，总计算量保持不变。这是一个优雅的性质：**多头注意力在不增加计算量的前提下获得了更强的表达能力**。

**与 RNN 的对比：**

RNN 逐个时间步处理，每个时间步需要 $$(d \times d) \cdot (d \times 1)$$ 矩阵-向量乘法，复杂度 $$O(d^2)$$，$$T$$ 步总计：

$$O(T \cdot d^2)$$

| 架构 | 计算复杂度 | 并行性 | 长序列瓶颈 |
|------|-----------|--------|----------|
| RNN | $$O(T \cdot d^2)$$ | 顺序依赖，无法并行 | 训练速度慢，但内存与 $$T$$ 线性 |
| 自注意力 | $$O(T^2 \cdot d + T \cdot d^2)$$ | 完全可并行 | $$T$$ 较大时 $$O(T^2)$$ 主导 |
| 多头注意力 | $$O(T^2 \cdot d + T \cdot d^2)$$ | 完全可并行 | 与单头注意力相同量级 |

**$$T$$ 与 $$d$$ 的相对重要性：** 在实际的 LLM 中，嵌入维度 $$d$$ 通常在 768--12288 范围，而序列长度 $$T$$ 在 1024（GPT-2）到 128K（GPT-4 Turbo）范围。当 $$T \ll d$$ 时，$$O(T \cdot d^2)$$ 项占主导（投影开销大）；当 $$T \gg d$$ 时，$$O(T^2 \cdot d)$$ 项占主导（注意力矩阵开销大）。现代长上下文模型面临的正是一种情况，这促使了 FlashAttention、Ring Attention 等高效实现的涌现。

## 3.8 代码实现一览

本章从零逐步构建了完整的注意力机制实现，每一步都在前一步基础上增加功能：

- **`SelfAttention_v1`**（3.4.2节）：最基本的可训练自注意力。使用 `nn.Parameter(torch.rand(...))` 手动定义 $$\mathbf{W}_q, \mathbf{W}_k, \mathbf{W}_v$$。权重使用均匀随机初始化，功能正确但非最优

- **`SelfAttention_v2`**（3.4.2节）：将手动参数替换为 `nn.Linear(d_in, d_out, bias=False)`。内部使用 Kaiming 初始化，训练更稳定。`nn.Linear` 以转置形式存储权重（`weight.shape = (d_out, d_in)`），在赋值给 `SelfAttention_v1` 时需要先转置

- **`CausalAttention`**（3.5.3节）：在自注意力的基础上加入两个关键增强：(1) 因果掩码——通过 `register_buffer` 存储上三角 $$-\infty$$ 掩码并在 softmax 前应用到注意力分数；(2) Dropout 层——在注意力权重上施加随机丢弃。支持批处理输入（三维张量 $$(B, T, d)$$）

- **`MultiHeadAttentionWrapper`**（3.6.1节）：通过 `nn.ModuleList` 堆叠多个 `CausalAttention` 实例，将各头的输出沿最后一维拼接。实现直观但每个头需要独立做线性投影，效率非最优

- **`MultiHeadAttention`**（3.6.2节）：高效的多头注意力实现——核心思想是"先做大投影再拆分"。所有头的 weight 矩阵被合并为一个大的 `nn.Linear(d_in, d_out)`，通过 `.view()` 和 `.transpose()` 将结果拆分为多头的形式，利用批处理矩阵乘法并行计算所有头的注意力。同时包含输出投影 `out_proj`

最终的 `MultiHeadAttention` 类将在第4章被集成为 GPT 模型 Transformer 块的核心组件，配合 Layer Normalization、前馈网络和残差连接一起构成完整的 LLM 架构。

## 3.9 要点总结

- **注意力机制**将输入元素转换为增强的上下文向量表示，融合了整个序列的信息。这是 Transformer 和 LLM 最核心的计算原语

- **自注意力**通过 Query、Key、Value 三个可训练投影（$$\mathbf{W}_q, \mathbf{W}_k, \mathbf{W}_v$$）计算加权的上下文向量：$$\mathbf{Z} = \text{softmax}(\mathbf{Q}\mathbf{K}^{\top}/\sqrt{d_k}) \mathbf{V}$$

- **缩放因子 $$\sqrt{d_k}$$** 是必要的——它将点积的方差从 $$d_k$$ 归一化到 1，防止 softmax 饱和导致的梯度消失。从方差分析严格推导：$$\text{Var}(\mathbf{q} \cdot \mathbf{k}) = d_k$$

- **Softmax** 是注意力机制的核心非线性激活：$$\text{softmax}(z_i) = e^{z_i} / \sum_j e^{z_j}$$。其雅可比矩阵为 $$\frac{\partial p_i}{\partial z_j} = p_i(\delta_{ij} - p_j)$$。当分布趋近 one-hot 时，所有梯度趋于 0——解释缩放因子的必要

- **因果掩码**通过在 softmax 前将未来位置的分数设为 $$-\infty$$，确保自回归生成中不存在信息泄漏。数学原理：$$\exp(-\infty) = 0$$，因此被掩码位置的注意力权重精确为零

- **Dropout** 以概率 $$p$$ 随机置零注意力权重，并按 $$1/(1-p)$$ 缩放保留值以维持期望不变。这是一种经典的正则化手段，仅在训练时启用

- **多头注意力**将输入拆分到 $$h$$ 个独立的表示子空间并行处理：每个头有独立的 $$\mathbf{W}_q^{(i)}, \mathbf{W}_k^{(i)}, \mathbf{W}_v^{(i)}$$，输出经拼接和 $$\mathbf{W}_O$$ 投影后融合：$$\text{MultiHead} = \text{Concat}(\text{head}_1, \ldots, \text{head}_h) \mathbf{W}_O$$

- **权重拆分法**是多头注意力的高效实现：使用统一的权重矩阵 + view/transpose 将一次大型矩阵乘法的结果拆分为多头，通过批处理并行计算所有头，避免了 for 循环带来的效率损失

- **计算复杂度**：自注意力为 $$O(T^2 \cdot d + T \cdot d^2)$$，对序列长度二次方但完全可并行。RNN 为 $$O(T \cdot d^2)$$，对序列长度线性但因顺序依赖无法并行。多头注意力与单头有相同的渐近复杂度——头数的增加不带来额外的计算开销

- **维度约束**：在 GPT 类模型中，输入嵌入维度 $$d_{\text{in}}$$ 与输出维度 $$d_{\text{out}}$$ 相同，这是构建深层网络时残差连接的前提条件（将在第4章详细展开）
